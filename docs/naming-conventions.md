# Naming Conventions

## Repositorio

- Usa nombres de repositorio cortos, descriptivos y aptos para publicación.
- Evita mezclar branding efímero con el nombre estable del catálogo.

## Skills

- El nombre visible debe ser legible por humanos: `Git & GitHub Expert`.
- El slug debe ser estable, en minúsculas y con guiones: `skill-git-github-expert`.
- El id debe ser único y con namespace: `codex.skill.git-github-expert`.

## Archivos obligatorios

- `SKILL.md`
- `prompt.md`
- `skill.json`
- `README.md`
- `CHANGELOG.md`

## Buenas prácticas

- Mantén el mismo slug en carpeta, catálogo y manifiesto.
- Evita abreviaturas crípticas.
- Si cambias el slug público, trátalo como cambio breaking.
