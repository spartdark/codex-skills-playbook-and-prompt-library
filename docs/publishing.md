# Publishing

## Objetivo

Publicar el repositorio como librería pública e instalable de skills de Codex, con releases repetibles y fáciles de compartir.

## Flujo recomendado

1. Trabaja en una rama descriptiva.
2. Ajusta contenidos y metadata.
3. Verifica coherencia entre `VERSION`, `catalog.json` y `skill.json`.
4. Crea un commit con Conventional Commits.
5. Abre un pull request.
6. Tras merge a `main`, crea un tag de release.
7. Publica la release en GitHub con notas claras.

## Qué incluir en una release

- versión del repo;
- resumen de skills añadidas o actualizadas;
- cambios rompientes si existen;
- instrucciones de instalación o actualización si cambian.

## Formato sugerido para release notes

- qué se añadió;
- qué se mejoró;
- qué cambió de forma incompatible;
- cómo actualizar.

## Distribución

Puedes compartir este repositorio de tres maneras:

- enlace al repositorio público;
- enlace a una release específica;
- archivo comprimido de una skill concreta para equipos internos.
