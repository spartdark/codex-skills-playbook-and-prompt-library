# Validation Checklist

- The problem is described clearly and separately from the solution.
- The target user and context are explicit.
- Critical assumptions are visible.
- The MVP is framed as an experiment, not just a smaller scope.
- There is traceability from problem to feature to metric.
- Context is separated into static knowledge, dynamic data, and user memory.
- Hallucination and context window risks are identified.
- Output rules and validation logic are concrete.
- Hypotheses, experiments, and falsification criteria are present.
- Technical and security risks are stated clearly.
