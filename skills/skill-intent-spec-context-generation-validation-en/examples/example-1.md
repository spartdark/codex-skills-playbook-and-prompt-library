# Example 1

## Request

"I have an idea for an AI operations assistant, but I do not want to build anything until we know the problem is real, the context model is clear, and the validation path is solid."

## Expected Behavior

- clarify the problem, user, urgency, and consequences;
- write a structured PRD with an MVP framed as an experiment;
- define static knowledge, dynamic data, and user memory;
- design validation experiments and falsification criteria;
- identify architecture and security risks before recommending action.
