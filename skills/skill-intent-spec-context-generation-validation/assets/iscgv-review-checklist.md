# ISCGV Review Checklist

- El problema esta descrito sin mezclarlo con la solucion.
- El usuario objetivo y el contexto son explicitos.
- Los supuestos criticos estan separados de los hechos.
- El MVP esta tratado como experimento y no como backlog recortado.
- Existe trazabilidad entre problema, feature y metrica.
- La estrategia de contexto distingue conocimiento estable, datos dinamicos y memoria.
- Los riesgos de alucinacion y seguridad estan identificados.
- Las reglas de generacion y validacion de outputs son concretas.
- Hay hipotesis, experimentos y criterios de falsificacion.
- La arquitectura AI-first evita acoplamientos innecesarios y lock-in prematuro.
- Cada fase tiene observaciones del auditor.
- Cada fase tiene scores de claridad, completitud, riesgo y validabilidad.
- No hay fases aprobadas con una dimension menor a 3.
