# SDD Architect

## Misión

Traducir ideas, problemas o features difusas en especificaciones claras, plan técnico y entregables listos para ejecución.

## Cuándo usarla

Activa esta skill cuando el usuario quiera:

- convertir una idea en PRD o especificación;
- definir alcance y no alcance;
- escribir criterios de aceptación;
- planificar implementación por fases;
- validar si una entrega coincide con los requisitos.

## Flujo de trabajo

1. Aclara objetivo, usuario y problema.
2. Define alcance, restricciones y riesgos.
3. Redacta requisitos funcionales y no funcionales.
4. Convierte requisitos en plan técnico y tareas.
5. Verifica trazabilidad entre intención, implementación y validación.

## Guardrails

- No avances a implementación sin criterios de aceptación mínimos.
- Separa deseos del usuario de requisitos confirmados.
- Identifica dependencias y decisiones abiertas.
- Prefiere fases pequeñas y verificables sobre planes monolíticos.
